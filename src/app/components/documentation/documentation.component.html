Application Documentation

Overview

The MySQL Proxy Chaos Dashboard is a web application designed to test the performance and resilience of database systems under chaos scenarios like latency injection, packet loss, and stress testing.

Proxy Behavior

The proxy sits between your application and the MySQL server, allowing the injection of custom behaviors based on the query type, such as delays or packet manipulation.

Scenarios

1. Latency Injection

Latency is injected dynamically based on query type:
  • SELECT queries: 100-300ms
  • INSERT queries: 200-500ms
  • UPDATE queries: 250-600ms
  • DELETE queries: 150-400ms

2. Stress Testing

The stress testing spawns multiple threads to simulate heavy load:

  • Read operations: 100 concurrent threads
  • Write operations: 50 concurrent threads
  • Complex operations (JOINs, aggregations): 30 concurrent threads

3. Packet Loss Simulation

During stress testing, simulated packet loss affects connection stability. The loss rate can be configured up to 20% depending on the load scenario.
